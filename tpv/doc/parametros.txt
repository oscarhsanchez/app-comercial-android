Parámetros

La tabla de parámetros como su nombre nos dice, obtiene las configuraciones de la TPV. 
Algunos parámetros son registrados desde del servidor y otros son registrados en local, 
por eso la tabla de parámetros necesita los siguientes campos:

* clave (text)
* valor (text)
* fecha (text) - guarda la fecha en la que se creó/se cambió este parámetro
* tipo (text) - "LOCAL" significa que es un parámetro creado por la aplicación. 
	"REMOTO" significa que es un parámetro creado por el servidor remoto.

Parámetros locales:

* databasestatus - Indica el estado de la base de datos. 
	La ausencia de este valor significa que la base de datos nunca fue creada.
	0 - base de datos creada pero no terminada, significa que todavía no está en estado funcional y que no se aceptan login de cache y el login normal necesita terminar la construcci√≥n de la base de datos para poder proseguir.
	1 - base de datos creada con Éxito, se permite login con cache y el login normal caso no haya cambiado de usuario asociado no necesita.
	El Registro se elimina cuando se resetea la base de datos, sea por cambio de usuario o por el motivo que sea.
	
	
* updatefrequency - Parametro que indica a cada cuanto tiempo se debe actualizar la base de datos.
					Su valor es devuelto en milisegundos.
					Este parametro puede ser local o remoto. En la ausencia del tipo remoto, o 
					en caso de que tenga un valor no correcto (fallo al convertir o menor que 3000) 
					se usa el valor local que por defecto vale 2 minutos (120000)
					
					
* tabletuser - Parámetro local que indica cual es el usuario asociado a la tableta.
